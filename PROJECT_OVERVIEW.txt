ENERGY PREDICTION SYSTEM - PROJECT OVERVIEW
================================================================================

SYSTEM SUMMARY
================================================================================

A distributed web application that:
1. Simulates IoT smart meters publishing energy data via MQTT
2. Uses a rule-based ML model to predict electricity prices
3. Logs predictions to a simulated blockchain for auditability
4. Provides a real-time React dashboard for users and admins

KEY FEATURES IMPLEMENTED
================================================================================

CORE:
  ✓ User registration & Authentication (JWT, BCrypt)
  ✓ Role-Based Access Control (Admin vs Regular User)
  ✓ House Management (CRUD operations)
  ✓ Real-time Energy Data Processing (MQTT)
  ✓ Price Prediction Algorithm (Decision Tree)
  ✓ Blockchain Logging (SHA-256 Hashing + Proof of Authority)
  ✓ React Frontend with Recharts data visualization
  ✓ 18 REST API Endpoints
  ✓ SQLite Database with GORM

ARCHITECTURAL HIGHLIGHTS:
  ✓ Hybrid Architecture (REST + Event-Driven)
  ✓ Microservices pattern (API Gateway + Simulator)
  ✓ Containerized with Docker Compose
  ✓ Secure Environment Configuration

DATABASE SCHEMA
================================================================================

1. users (authentication & profile)
2. households (properties owned by users)
3. predictions (energy readings + price analysis)
4. blockchain_log (immutable audit trail)
5. sqlite_sequence (internal)

API ENDPOINTS SUMMARY
================================================================================

Auth:
  POST /auth/register, /auth/login, /auth/logout, /auth/refresh

Data:
  GET /api/houses, POST /api/houses, PUT /api/houses/:id
  GET /api/predictions, GET /api/statistics
  GET /api/blockchain/verify/:hash

Admin:
  GET /admin/users, GET /admin/dashboard
